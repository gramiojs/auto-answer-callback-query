# Auto Answer Callback Query

GramIO plugin that automatically calls `answerCallbackQuery` after `callback_query` handlers if the handler didn't already answer.

## Project Structure

- `src/index.ts` — plugin source (single file)
- `src/index.test.ts` — tests
- `scripts/` — CI helpers (changelog generation, JSR publish prep)
- `dist/` — built output (generated by pkgroll)

## Commands

- `bun test` — run tests
- `bun run lint` — lint with biome
- `bun run lint:fix` — autofix lint issues
- `bunx pkgroll` — build for npm

## How the Plugin Works

The plugin registers middleware on the `callback_query` event via `Plugin.on()`. It wraps `context.answerCallbackQuery` with a proxy that tracks whether it was called. The auto-answer logic is inside `try/finally` around `await next()` — this ensures the callback query is answered even if the user's handler throws. API errors from the auto-answer are silently caught (the callback query may have expired).

`context.answer()` is an alias for `context.answerCallbackQuery()` in GramIO, so the proxy covers both methods.

## Publishing

Dual-published to npm (`@gramio/auto-answer-callback-query`) and JSR (`@gramio/auto-answer-callback-query`). The GitHub Actions workflow (`publish.yml`) handles both via `workflow_dispatch`.

## Testing

Tests use `@gramio/test` (TelegramTestEnvironment). After `click()`, a `setTimeout` tick is needed because the plugin's post-handler auto-answer runs asynchronously after the test environment resolves the update.
